Banking System (Web + Backend)

Inter-branch communication system with centralized control and operations

Project Overview

This project is a secure banking platform built using a Flask-based backend and SQL database. It supports a multi-branch architecture where a central zone (super admin) can manage and control all branch operations. The system includes account management, transactions, messaging between branches, and a centralized dashboard for monitoring activities.

Core Functionalities

  • Centralized control system for super admin to manage all branches.
  • Branch-level operations with role-based access control.
  • Secure authentication system for users and administrators.
  • Real-time communication between branches and central authority.

Account Management

  • Create and update customer accounts with details such as name, DOB, PAN, email, and balance.
  • Maintain structured records for each customer across branches.
  • Support multiple account types with validation rules.

Transactions

  • Deposit, withdrawal, and fund transfer operations.
  • Enforced business rules for account types:
    • Savings Account → Minimum balance ₹5000
    • Fixed Deposit → Minimum balance ₹10000
    • Current Account → No restrictions
  • Transaction history tracking for auditing and monitoring.

Messaging System

  • Internal messaging system for communication between branches and super admin.
  • Enables coordination, reporting, and issue resolution across the banking network.

Dashboard

  • Displays high-balance and low-balance accounts.
  • Shows distribution of different account types.
  • Provides transaction history and analytics.
  • Helps administrators monitor system performance and user activity.

Technologies Used

Python (Flask), SQL Database, HTML, CSS, JavaScript

RFID-Based Attendance System

Automated Attendance with RFID & Facial Recognition

Project Overview

This project is an automated attendance system that integrates RFID technology with facial recognition to identify students and mark attendance in real time. It eliminates manual attendance processes, improves accuracy, and ensures efficient tracking of student presence.

System Workflow

  • Each student is assigned a unique RFID card linked to their profile.
  • When a student scans the RFID card, the RFID reader captures the unique ID.
  • The captured data is processed using a Python backend.
  • The system communicates with Firebase to fetch student details in real time.
  • Student information is displayed on the screen automatically.
  • Attendance is marked as “Present” upon successful identification.
  • Facial recognition using OpenCV is used as an additional verification layer.

Key Features

  • Real-time attendance marking
  • RFID-based student identification
  • Facial recognition authentication (OpenCV)
  • Firebase real-time database integration
  • Automatic attendance logging
  • PDF report generation
  • Filtering by date, class, and student

Report Generation

The system allows teachers to generate and download attendance reports in PDF format. Reports can be customized based on:

  • Daily attendance records
  • Monthly summaries
  • Class-wise attendance
  • Individual student attendance using RFID

Technologies Used

HTML, CSS, JavaScript, Python, RFID Hardware, OpenCV, Firebase

Smart Poultry Farming System

Final Year B.Tech Capstone Project (IoT-Based Monitoring System)

Project Overview

This project is an IoT-based Smart Poultry Farming system designed to monitor environmental conditions such as temperature, ammonia (NH₃), carbon dioxide (CO₂), and smoke levels in real time. The system enables remote monitoring of poultry farm conditions through a web-based dashboard, eliminating the need for continuous physical presence.

System Architecture & Working

  • IoT sensors are deployed in the poultry environment to capture environmental parameters.
  • NodeMCU (WiFi-enabled microcontroller) collects sensor data.
  • The collected data is transmitted over WiFi to a backend/database in real time.
  • The database stores incoming sensor readings for processing and visualization.
  • A web-based dashboard retrieves data via REST API and displays live farm conditions.
  • Admin can monitor the poultry environment remotely without physical presence.

Features

  • Real-time monitoring of temperature, NH₃, CO₂, and smoke levels
  • Wireless data transmission using NodeMCU (WiFi)
  • Centralized database storage for sensor data
  • Web-based dashboard for live visualization
  • Remote access for farm monitoring and decision-making
  • Scalable IoT architecture for environmental tracking

Research & Academic Contribution

  • Published at the 7th International Joint Conference on Computing Sciences (ICCS-2023) — “KILBY100”.
  • Focused on integrating IoT technologies for environmental monitoring in poultry farming operations.
  • Designed and developed a web-based dashboard for real-time visualization of sensor data.
  • Implemented Arduino/NodeMCU-based hardware integration for sensor communication.
  • Contributed to end-to-end system development including hardware setup, firmware coding, backend integration, and deployment.
  • Authored a thesis paper documenting system design, implementation, and results.

Key Benefits

  • Enables remote monitoring of poultry farm conditions
  • Reduces manual labor and supervision effort
  • Improves farm management efficiency
  • Provides real-time insights for better decision-making
  • Enhances environmental safety for livestock

Technologies Used

Python, IoT Sensors, NodeMCU, REST API, Web Dashboard, Wireless Communication

Automatic Hydroponics System

IoT-Based Smart Agriculture Solution

Project Overview

This project is an IoT-based Automatic Hydroponics System designed to grow plants without soil by delivering nutrients directly through water. The system uses sensors and microcontrollers to monitor environmental and chemical parameters and automatically regulate nutrient levels for optimal plant growth.

System Concept

  • Plants are grown in a controlled water-based environment instead of soil.
  • Sensors continuously monitor key parameters such as pH levels and nutrient concentration.
  • Based on sensor readings, the system automatically adjusts nutrient (NPK) levels in the water.
  • Microcontroller (NodeMCU/Arduino) processes sensor data and controls actuators.
  • IoT connectivity enables remote monitoring and system integration.

Functional Workflow

  • Sensors collect real-time data from the hydroponic setup (pH, nutrients, etc.).
  • Data is processed by the microcontroller (Arduino/NodeMCU).
  • The system determines whether nutrient levels need adjustment.
  • If required, pumps/dispensers automatically add NPK nutrients into the water.
  • System ensures optimal conditions for plant growth without manual intervention.
  • Data can be transmitted via IoT for monitoring and logging.

Collaboration

This project was developed in collaboration with an agriculture student, combining technical expertise with domain knowledge in plant science to optimize nutrient delivery and plant growth conditions.

Key Features

  • Soilless plant cultivation using hydroponics
  • Automatic nutrient regulation based on sensor input
  • pH monitoring for maintaining water quality
  • Automated NPK nutrient delivery system
  • IoT-enabled monitoring and control
  • Reduced manual intervention in farming

Key Benefits

  • Improves crop yield and growth efficiency
  • Reduces water and resource consumption
  • Enables precision agriculture techniques
  • Minimizes human effort and errors
  • Supports sustainable farming practices

Technologies Used

NodeMCU, Arduino, Sensors (pH, nutrient), IoT Communication, Python
